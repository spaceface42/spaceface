Intrinsic layout

This page uses intrinsic, constraint-based CSS. Instead of defining fixed layouts for specific screen sizes, the elements describe how they want to flow, wrap, and share space. The browser decides the final layout based on the available room, keeping the design fluid, simple, and less prescriptive.

I’d call it constraint-based CSS or intrinsic layout. More natural phrases for the style you mean: intrinsic web design: let content, available space, and browser layout rules decide more of the behavior fluid layout: adapts continuously instead of jumping at fixed breakpoints constraint-based styling: define gentle rules like min size, wrapping, and flow instead of exact layouts less prescriptive CSS: describe intent without locking every viewport state content-first layout: the content’s natural size drives the layout progressive CSS: start with simple natural behavior, add rules only when needed For your footer example, the principle is basically: “Set relationships and constraints, not exact device-specific layouts.” So you might say: “I prefer intrinsic, constraint-based CSS over breakpoint-heavy, prescriptive CSS.”
